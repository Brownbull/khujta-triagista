[2026-04-07 16:26:57] sebastianmontagna_54230: FAQ #6: Should I use specific API providers? Will you provide test API keys?

We will not provide API keys for testing. Your .env.example file should include clear placeholders and comments for every environment variable your application needs — but never actual keys.

‼️ There is no restriction on which LLM provider you use. You are free to choose OpenAI, Anthropic, Google Gemini, OpenRouter, Groq, Mistral, or any other provider that fits your solution.

That said, keep the following in mind:

🔻 Be explicit about which provider and model your project requires. Document this clearly in your QUICKGUIDE.md and .env.example so reviewers can set it up quickly.

🔻 OpenAI-compatible APIs are a plus. If your application supports the OpenAI-compatible API format (which providers like OpenRouter, Groq, and many others also support), it makes it easier to swap providers during testing. 

From a software engineering perspective, this is better for maintainability, scalability, and portability.

✨ OpenRouter is a good option if you want to give reviewers flexibility. A single API key can access multiple models from different providers.
