📝 What happened?

Payment gateway returning HTTP 502 errors during checkout. All customers affected since 14:30 UTC. Stripe webhook handler appears to be timing out. Error rate jumped from 0.1% to 45%. Checkout completion rate dropped to near zero.

✅ Injection: 2% ✅ Clean

💡 Root Cause Analysis

The Stripe webhook endpoint is experiencing upstream latency causing timeout cascade. The payment gateway connection pool may be exhausted due to retries on failed webhook deliveries.

📂 Related Files

spree/payment.rb spree/payment_method.rb api/payments_controller.rb spree/order.rb

🎯 Recommended Actions

1

Check Stripe API status page and verify webhook endpoint health via stripe trigger CLI.

2

Increase connection pool size and timeout thresholds in StripeCreditCard payment method config.

3

Enable circuit breaker on the webhook handler to stop retry storms and let the pool recover.

4

Deploy a fallback payment flow that queues orders for retry when gateway is unhealthy.

5

Add alerting on connection pool exhaustion and webhook latency p99 to prevent recurrence.
